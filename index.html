Configuração do evento

Metadados do evento usados para emitir certificados e popular a app de presenças. Guardado em data/evento.json (público — sem PII).

Inteiro único. Usado na ligação inscritos ↔ presenças.
Sequencial do ano corrente. Auto-incrementa a cada emissão.

Programa do evento

Agenda científica do evento. Visível para participantes (página do certificado) e para o staff (menu do tablet). Pode ser alterada em tempo real — basta clicar Guardar configuração no fim. As alterações aparecem nos tablets no próximo poll.

B · Operação no terreno

Acessos Staff (até 5 contas)

Cada operador no balcão faz login com Utilizador + Palavra-passe. As palavras-passe são guardadas como hash SHA-256 em evento.json — o texto original nunca é gravado. Deixar campos vazios para desactivar um slot.

Operadores escolhem o nome no dropdown no tablet e introduzem só a sua palavra-passe. A autorização do tablet (PAT do GitHub) é automática — o tablet decifra-o no boot a partir da password admin embutida.

Apreciação · Microsoft Forms

URL do questionário Microsoft Forms aplicado presencialmente no balcão de saída. Lido via código QR, com pré-preenchimento da identificação do participante.

Placeholders suportados: {id}, {nome}, {cargo}, {entidade}. No portal Microsoft Forms: Collect responses → Get a link with pre-filled answers. Substitui os valores das perguntas pelos placeholders na URL copiada.
C · Certificação e envio

Assinatura do certificado

Sem imagem carregada. O envio de certificados fica bloqueado até carregar a assinatura.
Duas opções: Assinar agora (com rato, dedo ou caneta táctil no próprio admin) ou Carregar PNG (ficheiro pré-feito). PNG ideal: fundo transparente, ~400px largura. Persiste em evento.json e replica para assets/assinatura-comandante.png no repo Certificados. Sem assinatura carregada, o envio de certificados fica desactivado.

Email — remetente, CC e corpo

Placeholders: {{Nome}}, {{Titulo}}, {{NumeroCertificado}}, {{DataEvento}}.
Suporta os mesmos placeholders + {{Cargo}}, {{Entidade}}, {{Link}}, {{Local}}, {{HoraInicio}}, {{HoraFim}}, {{CargaHoraria}}, {{Signatario}}, {{SignatarioCargo}}, {{AssinaturaImagem}}.

D · Segurança

SECRET do hashing de certificados

Chave secreta usada no hash dos certificados. Tem de bater exactamente com a constante SECRET em Certificados/index.html.

🔒 Privado por design: a SECRET é guardada apenas no sessionStorage deste browser, nunca é publicada no GitHub. Cada admin tem de a introduzir uma vez por sessão.
Apaga ao fechar o separador. Para a guardar de novo, voltas a preencher e clicas em "Guardar configuração" (mas só altera a tua sessão local).
⚠️ Mudar esta chave invalida todos os certificados emitidos antes da mudança.

Segurança dos tablets

Em caso de tablet perdido / roubado / fim do evento, esta acção desautoriza todos os tablets ao mesmo tempo. Cada tablet, no próximo poll, faz logout e volta ao ecrã de password. Não revoga o PAT em si — quem tiver acesso ao código continua a ter — mas força reconfiguração imediata. Para revogar o PAT é preciso rodá-lo no GitHub e regenerar o cifrado.

E · Ciclo de vida — próximo evento

Arquivar evento actual e preparar o próximo

No fim de cada evento (depois de enviados todos os certificados), arquiva o estado actual e prepara a app para o próximo. Não apaga nada — o snapshot é guardado num branch arquivo (ex: arquivo/2026-05-18) e podes voltar a consultá-lo a qualquer momento.

Workflow recomendado:
  1. Enviar todos os certificados (tab Envio)
  2. 💾 Backup no header (snapshot ZIP local)
  3. Rodar o PAT do GitHub e regenerar o cifrado (Eventos/tools/cifrar-pat.html)
  4. Botão abaixo: cria branch arquivo/<data> e reseta presenças/entregas/certificados/apreciação
  5. Preencher os campos acima para o próximo evento (ID, data, título, programa) → Guardar configuração
  6. Importar lista de inscritos nova no tab Inscritos

Importar lista de inscritos

Faz upload do ficheiro Excel. As colunas Nome / Email / Cargo / Entidade / Categoria são auto-detectadas. Os emails ficam apenas no teu browser (sessionStorage) — nunca são publicados no GitHub.

Clica numa linha para editar · clica no pill de estado para ciclar

Lista pública

Estado actual de data/inscritos.json no GitHub. Schema inscritos@2 inclui token, email e autorizaContacto.

⚠️ Existem alterações não publicadas. Clica "Publicar inscritos.json" para enviar para o GitHub.

Envio de certificados por email

Gera ficheiros .eml para cada certificado e empacota em zip. Depois arrasta os ficheiros para a pasta Drafts do Outlook desktop e selecciona "Enviar tudo".

Como funciona: 1) carrega o Excel na tab Inscritos para teres os emails em sessão; 2) clica "Gerar zip"; 3) extrai o zip; 4) selecciona todos os .eml e arrasta para Outlook → Drafts; 5) Selecciona todas as drafts e Send.

Acesso ao admin

Introduz a password admin. O token GitHub é descodificado em segundos a partir do PAT cifrado embutido no código.
